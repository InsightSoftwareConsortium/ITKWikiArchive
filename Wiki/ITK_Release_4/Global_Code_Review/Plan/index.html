Plan

The global code review will proceed according to the following suggested plan.

  1. Partition the toolkit modules into six large groups
  2. Assign each group to one of the major contractors in ITKv4
  3. Each contractor will code review the files that have been assigned
    • Observations on these review will be collected in a parallel Git repository
      • https://github.com/InsightSoftwareConsortium/itk-retroactive-review
    • Minor issues will be addressed directly during the review
    • Major issues will be entered as Bugs, and will be discussed in the developers list.
